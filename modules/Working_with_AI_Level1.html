📌 Module 1: 왜 AI를 써도 퇴근은 늦어질까?

① Experiment — 오늘 해보세요

당신이 해야 할 일: 아래 작업을 직접 해보고, 시간을 재보세요.

  1. 회사 소개를 1페이지로 정리하세요. (예: 회사명, 설립일, 주요 사업, 핵심 가치)
  2. AI(챗GPT, Claude 등)의 새 채팅창을 엽니다.
  3. 방금 정리한 회사 소개를 복사해서 붙여넣기 한 후, 이렇게 물어보세요: "우리 회사에 대해 간단히 요약해줘."
  4. 잠시 후, 완전히 새로운 채팅창을 엽니다.
  5. 똑같은 회사 소개를 다시 복사해서 붙여넣기 하고, 똑같은 질문을 합니다.
  6. 이 과정(새 채팅창 열기 → 복붙 → 질문)을 오늘 하루 동안 3번만 반복해보세요.
  7. 총 몇 분이 소요되었나요?

② Reflection — 생각해보기

  • 오늘 같은 회사 소개를 3번이나 복사해서 붙여넣으면서, 어떤 기분이 들었나요?
  • 내일도 이렇게 3번, 모레도 또 3번 해야 한다면, 1년이면 몇 시간을 낭비하게 될까요? (계산: 3분 × 3회 × 5일 × 52주 = 39시간)
  • AI는 분명 똑똑한데, 왜 우리 회사 소개를 절대 외우지 못할까요?

③ Discovery — 방금 깨달은 것

"아, AI는 내가 알려주지 않으면 나를 절대 기억하지 못하는구나."

④ Rule — 법칙

법칙내용
Rule 1AI의 기억은 휘발성이다. AI의 대화창(Context)은 RAM과 같습니다. 세션이 끝나면 모든 것이 사라집니다.
Rule 2반복 설명은 생산성 킬러다. 같은 내용을 매일 반복해서 설명하는 데 드는 시간은 누적되면 막대합니다.
Rule 3기억은 파일로 외부화한다. 한 번 작성한 내용을 파일(Storage)에 저장하면, AI는 언제든지 그 파일을 읽을 수 있습니다.

⑤ Practice — 오늘부터 실천

  1. 메모장(또는 VS Code)을 열고 company_intro.md 파일을 만드세요.
  2. 방금 복붙했던 회사 소개를 그 파일에 붙여넣고 저장하세요.
  3. 이제 AI에게 이렇게 말해보세요: "company_intro.md 파일을 읽고, 우리 회사를 한 줄로 소개해줘."
  4. AI가 파일을 읽고 정확히 답변하는지 확인하세요.

🎯 이제 당신은 매일 3분씩 낭비하던 복붙 시간을 0초로 만들었습니다!

⑥ Common Mistakes — 하지 마세요

❌ 하지 마세요✅ 이렇게 하세요
"AI가 한 번 알려주면 외울 거야.""AI는 기억하지 못한다. 파일을 만들어라."
"파일에 적는 게 더 귀찮아.""오늘 5분 투자로 1년 39시간을 절약하라."

📌 Module 2: 업무를 분해해보자

AI를 쓰기 전에, 당신의 업무를 분해하는 법을 먼저 배웁니다.

① Experiment — 당신의 업무를 쪼개보세요

당신이 해야 할 일: 오늘 한 업무 하나를 골라, 종이에 적어보세요.

  1. 이 업무를 시작하기 전에, 무엇이 필요했나요? (재료)
  2. 그 재료들을 가지고 무슨 일을 했나요? (가공)
  3. 그 결과로 무엇이 나왔나요? (결과물)

② Reflection — 생각해보기

  • 위 3가지 중, 가장 많은 시간을 쓴 단계는 어디인가요?
  • 만약 2번(가공)이 단순하고 반복적이었다면, 누군가가 대신 해줄 수 있을까요?

③ Discovery — 방금 깨달은 것

"아, 내가 하는 모든 업무는 재료 → 가공 → 결과물로 나눌 수 있구나."

④ Rule — 법칙

법칙내용
Rule 4모든 업무는 3단계로 분해된다. 재료 → 가공 → 결과물. 이 틀에 맞지 않는 업무는 없습니다.
Rule 5AI는 '가공'을 잘한다. 요약, 분류, 변환, 초안 작성은 AI의 특기입니다.
Rule 6사람은 '재료 선정'과 '검증'을 잘해야 한다. AI에게 무엇을 넣을지, 결과가 맞는지는 사람의 몫입니다.

⑥ Common Mistakes — 하지 마세요

❌ 하지 마세요✅ 이렇게 하세요
"AI야, 보고서 만들어줘." (재료 없이 결과물만 요청)"이 재료를 읽고, 이렇게 가공해줘."

📌 Module 3: AI의 사무실을 만들자

Module 1에서 만든 AI_Work 폴더를 진짜 '사무실'처럼 꾸며봅시다.

① Experiment — 사무실을 비교해보세요

상황 A: 서류가 여기저기 흩어져 있고, 파일명이 aaaa.md, 최종_final.md, new.md입니다.
상황 B: 모든 서류는 폴더에 정리되어 있고, 파일명은 260708_프로젝트-내용.md처럼 명확합니다.

질문: 당신의 AI_Work 폴더는 A와 B 중 어디에 더 가깝나요?

② Reflection — 생각해보기

  • AI가 파일을 찾기 위해 폴더를 뒤적거려야 한다면, 얼마나 비효율적일까요?
  • 파일명이 정리되지 않으면, AI(그리고 미래의 당신)가 원하는 파일을 찾을 수 있을까요?

③ Discovery — 방금 깨달은 것

"아, AI에게 파일을 읽으라고 시키려면, 내가 먼저 파일을 잘 정리해둬야 하는구나."

④ Rule — 법칙

법칙내용
Rule 7AI_Work는 AI의 사무실이다. AI가 일하는 모든 파일은 이 폴더 안에 있어야 합니다.
Rule 8파일명은 규칙적으로. YYMMDD_프로젝트-내용.md 형식을 사용하세요.
Rule 9덮어쓰기 금지, 버전 생성. 수정 시 v1.0 → v1.1로 새 파일을 만드세요.

⑤ Practice — 사무실 인테리어

📌 Module 4: AI를 채용하자 (취업 규칙)

회사에도 취업규칙이 있듯이, AI에게도 업무 규칙이 필요합니다.

① Experiment — 규칙이 없는 AI에게 일을 시켜보세요

  1. AI에게 말합니다: "이번 주 고객 응대 내역을 정리해줘."
  2. AI가 데이터를 정리하기 시작합니다.
  3. 이번에는 이렇게 말해보세요: "그런데, 방금 한 작업은 절대 해서 안 되는 작업이었어."

→ AI는 당황하지 않고, 그냥 "죄송합니다. 다음에는 그렇게 하지 않겠습니다."라고 답합니다. AI는 방금 실수에 대해 전혀 학습하지 못했습니다.

② Reflection — 생각해보기

  • AI가 무엇을 해도 되는지, 하면 안 되는지 미리 알려주지 않으면, AI는 실수할 수밖에 없습니다.
  • 실수할 때마다 "그거 하지마"라고 말하는 것은, 인턴에게 매일 업무 규칙을 알려주는 것과 같습니다. 비효율적이지 않나요?

③ Discovery — 방금 깨달은 것

"아, AI에게는 사전에 명확한 업무 규칙을 줘야 하는구나."

④ Rule — 법칙

법칙내용
Rule 10규칙은 사전에 정의한다. AI는 '실수'를 학습하지 않습니다. 미리 경계를 알려줘야 합니다.
Rule 11규칙은 파일에 저장한다. 매번 말하지 말고, rules/AI_RULES.md에 적어두세요.
Rule 12규칙은 3가지로 구분한다. ① 자율 허용, ② 보고 필요, ③ 절대 금지

⑤ Practice — 취업 규칙 작성하기

  1. AI_Work/rules/AI_RULES.md 파일을 만드세요.
  2. 아래 내용을 채워넣으세요: 
    ```markdown
# AI 취업 규칙 (v1.0)

## 1. 당신의 정체성
당신은 내 업무를 돕는 AI 비서입니다.

## 2. 자율 허용 (스스로 해도 되는 일)
- 문서/코드 초안 작성
- 공개된 자료 수집 및 요약
- 파일명 정리, 오타 수정

## 3. 보고 필요 (반드시 내게 물어볼 것)
- 고객에게 보낼 최종 문서
- 비용(결제, 구매)이 발생하는 작업
- 실제 서버에 배포하는 작업

## 4. 절대 금지 (절대 하지 말 것)
- 내 계정/비밀번호/API 키를 저장하거나 출력
- 실제 고객 데이터를 외부로 전송
- 내 승인 없이 외부에 공유
```
  3. AI에게 말하세요: "rules/AI_RULES.md를 읽고, 당신이 자율로 할 수 있는 일과 절대 하면 안 되는 일을 요약해줘."

⑥ Common Mistakes — 하지 마세요

❌ 하지 마세요✅ 이렇게 하세요
규칙을 만들었지만, AI에게 읽으라고 안 시킨다.대화 시작 시 "AI_RULES.md를 읽고 시작해줘"라고 말한다.
규칙이 너무 추상적이다. ("잘해줘", "적절히 해줘")구체적인 예시를 포함한다.

📌 Module 5: 품질을 확인하자 (QC)

AI는 빠르지만, 정확하지 않을 수 있습니다. 품질을 확인하는 것은 오직 사람의 몫입니다.

① Experiment — AI를 속여보세요

  1. AI에게 말합니다: "내 업무는 철저한 사실 확인이 중요해. 거짓 정보는 절대 쓰면 안 돼."
  2. 이어서 말합니다: "그런데, 오늘 점심 회의 시간을 1시라고 해줘. (실제는 12시)"

→ AI는 모순을 인지하지 못하고, "네, 1시로 하겠습니다"라고 답하며 잘못된 정보를 그대로 사용할 수 있습니다.

② Reflection — 생각해보기

  • AI는 왜 모순을 감지하지 못했을까요? (AI는 '사실'과 '거짓'을 판단하지 않습니다.)
  • 만약 이 잘못된 회의 시간이 팀원들에게 공유되었다면, 누가 책임을 져야 할까요? (AI가 아니라, 당신입니다.)

③ Discovery — 방금 깨달은 것

"아, AI는 진실을 모르는구나. 그냥 그럴듯한 답변을 만들어낼 뿐이야."

④ Rule — 법칙

법칙내용
Rule 13AI는 진실을 모른다. 그럴듯한 답변을 생성할 뿐입니다.
Rule 14검증은 사람의 몫이다. AI가 만들었어도, 당신이 책임집니다.
Rule 153단계 QC를 실행하라. 기계 → 논리 → 인간 순서로 검증하세요.

3단계 QC 공식

단계방법질문

② 논리 검증내가 직접 사고하며 확인날짜/숫자가 일관된가? 모순은 없는가?
③ 인간 검증최종 사용자 관점이 결과물을 고객에게 보여줘도 되나?

⑤ Practice — QC 체크리스트 만들기

  1. AI_Work/rules/QC_checklist.md 파일을 만드세요.
  2. 아래 내용을 채우세요: 
    ```markdown
# 품질관리(QC) 체크리스트

## ① 기계 검증
☐ 파일이 정상적으로 저장되었는가?
☐ 프로그램 오류(빨간색 에러)는 없는가?

## ② 논리 검증
☐ 숫자와 날짜가 일관된가?
☐ 주장하는 내용의 근거가 타당한가?
☐ 빠진 내용이나 모순되는 내용은 없는가?

## ③ 인간 검증
☐ 이 결과물을 실제 업무에 사용해도 안전한가?
☐ 내가 이 내용에 대해 책임질 수 있는가?
```

📌 Module 6: 반복을 레시피로 바꾸자 (스킬 만들기)

자주 하는 요청은 레시피(템플릿)로 만들어서, 명령어 하나로 실행합시다.

① Experiment — 반복되는 질문을 찾아보세요

당신이 해야 할 일: 지난 한 주 동안, 당신이 AI에게 가장 자주 시킨 요청 3가지를 적어보세요.

  1. [예: 이메일 초안 작성]
  2. [예: 회의록 요약]
  3. [예: 보고서 양식 생성]

질문: 이 3가지 요청을 할 때마다, 당신은 같은 설명을 반복하고 있지 않나요?

② Reflection — 생각해보기

  • 매번 "포맷은 이렇게 해줘", "분량은 3페이지", "말투는 격식 있게"와 같은 설명을 반복한다면, 얼마나 많은 시간이 낭비될까요?
  • 만약 이 요청을 한 단어 명령어로 실행할 수 있다면, 얼마나 삶이 편해질까요?
